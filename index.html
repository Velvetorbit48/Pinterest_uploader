Automate publishing to Pinterest — securely and reliably

PinFlow is a lightweight, developer-focused uploader that streamlines how creators and small teams publish Pins. OAuth-driven authorization, scheduled bulk uploads, templated descriptions and automatic image optimization make repetitive workflows dependable and repeatable.

Who this is for

PinFlow is built for independent creators, marketers and small agencies who need predictable, repeatable Pin publishing without complex infrastructure. It is ideal when you:

  • Manage dozens or hundreds of Pins per month
  • Want schedule-based publishing and template-driven content
  • Prefer ownership of your assets and a clear audit trail

Core features

  • Scheduling
    Timezone-aware publish scheduling
  • Templates

  • Image optimization
    Auto-format/resize for best Pinterest rendering
  • Audit logs

Quick start

  1. Create a GitHub repository and add this index.html as your site root.
  2. Enable GitHub Pages on the repository (Branch: main, folder: /).
  3. Register a Pinterest Developer app; set your site URL & redirect URI to your GitHub Pages URL.
  4. Use the OAuth flow to obtain a long-lived token for your account or team.

How PinFlow works — a concise overview

PinFlow is intentionally minimalist: the landing page acts as the public redirect destination and verification endpoint for Pinterest's developer tools while the uploader logic runs independently (self-hosted scripts, CI jobs, or simple serverless functions). This keeps the surface area small and easy to verify for Pinterest's app review process.

1. Authorize

Users complete Pinterest OAuth in a secure browser session. PinFlow stores only the OAuth token and minimal metadata required to publish on behalf of the user.

2. Prepare

Create pin templates, import a CSV of items or upload images. Use the UI or a developer script to preprocess images and descriptions.

3. Publish

Schedule or trigger uploads. PinFlow performs API calls to Pinterest to create Pins and logs each action for traceability.

Privacy & data handling (summary)

We prioritize minimal data collection and transparent handling. Below is a plain-language summary of what a GitHub Pages + PinFlow deployment typically collects and how it is used. This is a sample policy — adapt it for your jurisdiction and consult legal counsel for production use.

Data we collect

  • OAuth tokens: Stored to authenticate with Pinterest on behalf of an account. Tokens are encrypted at rest.
  • Pin metadata: Titles, descriptions, destination URLs, publish schedule and tags to build the Pin request payload.
  • Uploaded media: Images you choose to publish. If you host media in third-party storage, we store only references/URLs.
  • Audit logs: Non-sensitive operational records showing who initiated publishes and when.

How we use data

Data is used exclusively to create and schedule Pins as you instruct. We do not sell personal data. Tokens are used only to access Pinterest API endpoints required for publishing and are revoked upon user request.

Third-party services

PinFlow leverages the Pinterest API and optionally image storage or CDN services chosen by you. When enabling analytics or other integrations, those services may collect additional usage data — make sure to disclose those services in your final privacy policy.

User rights & contact

You may request token revocation, data deletion or a copy of your data by contacting the site administrator at you@yourdomain.com. Implement these requests promptly to stay compliant with applicable local laws.

Frequently asked questions

Do I need a company website to use the Pinterest API?

Pinterest typically requires a verifiable website and a clear privacy policy for developer app registration. A GitHub Pages site with a visible Impressum and privacy policy is accepted in most cases for initial verification.

Is PinFlow storing my Pinterest password?

No. PinFlow uses the OAuth flow and stores only tokens issued by Pinterest. You retain control and can revoke access at any time through Pinterest or by contacting the site operator.

Is this official Pinterest software?

No — this is an independent uploader built to work with Pinterest's developer API. Always review Pinterest's Terms of Service before automating account actions.

Get started — deployment checklist

  1. Create a new GitHub repository and add this index.html as the site root.
  2. Push to the main branch and enable GitHub Pages under Settings → Pages (source: main / root).
  3. Register a Pinterest Developer application and set the website URL to your GitHub Pages URL (for example: https://your-username.github.io/your-repo/).
  4. Configure the OAuth redirect URI to the same GitHub Pages URL (append a path if required, e.g., /auth/callback).
  5. Test the OAuth flow locally and in production — ensure tokens are encrypted in storage and revoke tokens in Pinterest to test cleanup.
